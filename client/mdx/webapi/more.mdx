## Progressive Web Apps and Service Workers

PWA는 웹과 네이티브 앱의 기능 모두의 이점을 갖도록 수 많은 특정 기술과 표준
패턴을 사용해 개발된 웹 앱입니다.

https://developer.mozilla.org/ko/docs/Web/Progressive_web_apps/Tutorials/js13kGames

다음은 웹 앱을 PWA로 식별되기 위해 확인해야 하는 몇 가지 핵심 원칙입니다.

- 발견 가능, 컨텐츠를 검색 엔진을 통해 찾을 수 있습니다.
- 설치 가능, 기기의 홈 화면에서 사용할 수 있습니다.
- 연결 가능, 간단하게 URL을 전송해 공유할 수 있습니다.
- 네트워크 독립적, 오프라인이나 불안한 네트워크 연결에서 동작합니다.
- 점진적, 최신 브라우저의 모든 기능은 사용할 수 없지만 이전 브라우저의 기본
  기능은 여전히 사용할 수 있습니다.
- 재 참여 가능(Re-engageable), 새 컨텐츠가 사용 가능할 때마다 알림을 보낼 수
  있습니다.
- 반응형, 모든 기기의 화면이나 브라우저에서 사용할 수 있습니다 — 모바일 폰,
  태블릿, 노트북, TV, 냉장고, 등.
- 안전, 여러분과 앱 사이의 연결이 여러분의 민감한 데이터에 접근하려는 모든
  제3자로부터 안전합니다.

**ServiceWorker**는 웹 앱의 네트워크 요청을 가로채 캐시등의 처리를 할 수
있습니다. 등록된 service worker는 브라우저의 local storage에 계속 존재하고
사이트 재방문시 재활성화됩니다.

https://developer.mozilla.org/ko/docs/Web/API/Service_Worker_API

https://serviceworke.rs/

`Cache` API로 Request Response 객체를 캐싱할 수 있습니다. Service worker뿐만
아니라 밖에서도 사용할 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Cache

Web app manifest를 추가하면 장치의 홈 화면에 웹사이트를 추가할 수 있습니다.

https://developer.mozilla.org/ko/docs/Web/Manifest

`Notification` API로 노티를 기기에 보낼 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Notification

`Push` API로 service worker가 서버를 구독하게하여 웹 앱이 실행중이지 않을 떄도
알림을 보낼 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Push_API

## Mobile Device APIs

Geolocation API로 유저의 위치를 받을 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Geolocation

iOS를 제외하고, `navigator.vibrate()`로 모바일 기기가 진동하게 할 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Vibration_API

`ScreenOrientation` API로 기기의 방향을 알 수 있습니다.

```js
screen.orientation.addEventListener('change', (event) => {
  const type = event.target.type;
  const angle = event.target.angle;
  console.log(`ScreenOrientation change: ${type}, ${angle} degrees.`);
});
```

https://developer.mozilla.org/en-US/docs/Web/API/Screen_Orientation_API

`devicemotion`과 `deviceorientation` 이벤트도 있습니다:

```js
addEventListener('devicemotion', (e) => {
  let { x, y, z } = e.accelerationIncludingGravity;
  console.log(x, y, z);
});
```

https://developer.mozilla.org/en-US/docs/Web/API/Window/devicemotion_event

안드로이드 크롬에서는 `Sensor` API를 사용할 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Sensor_APIs

## Binary APIs

JS에서 바로 로컬 파일을 읽을 수는 없고, 유저가 form등을 통해 업로드한 파일을
`File` 객체로 얻을 수 있습니다. `File`은 `Blob`을 상속합니다.

`TextEncoder`와 `TextDecoder`를 통해 이진 데이터와 텍스트 사이를 오고갈 수
있습니다. 스트림도 지원합니다.

https://developer.mozilla.org/en-US/docs/Web/API/TextEncoder

https://developer.mozilla.org/en-US/docs/Web/API/TextDecoder

## Media APIs

`navigator.mediaDevices.getUserMedia()`로 유저의 마이크나 카메라에 접근할 수
있습니다. 반환된 `MediaStream` 객체를 `<video>`의 `srcObject`에 넣어 유저에게
보여주거나 `MediaRecorder`객체로 저장할 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getUserMedia

https://developer.mozilla.org/en-US/docs/Web/API/MediaStream

https://developer.mozilla.org/en-US/docs/Web/API/MediaRecorder

WebRTC(Web Real-Time Communication)은 웹 애플리케이션과 사이트가 중간자 없이
브라우저 간에 오디오나 영상 미디어를 포착하고 마음대로 스트림할 뿐 아니라,
임의의 데이터도 교환할 수 있도록 하는 기술입니다.

https://developer.mozilla.org/ko/docs/Web/API/WebRTC_API

## Cryptography and Related APIs

```js
let arr = new Int8Array(10);
crypto.getRandomValues(arr);
console.log(...arr);

// 암호화 관련 함수를 제대로 쓰는 것은 어렵고
// 대부분의 경우 직접 쓸 일이 없기에 이름이 subtle입니다.
console.log(crypto.subtle);
```

**Credential Managment API**로 공개키를 생성하고 저장하고 가져올 수 있습니다.

https://developer.mozilla.org/en-US/docs/Web/API/Credential_Management_API

**Web Authentication API**는 위 API의 확장입니다.

https://developer.mozilla.org/en-US/docs/Web/API/Web_Authentication_API

**Payment Request API**는 웹에서 신용 카드 결제를 지원합니다. 신용 카드 정보를
안전하게 저장합니다.

https://developer.mozilla.org/en-US/docs/Web/API/Payment_Request_API
